# ⚽ World Cup RAG Agent

A conversational AI assistant built on FIFA World Cup historical data using 
a RAG (Retrieval Augmented Generation) pipeline. Built two ways — 
no-code using n8n and production Python implementation.

---

## 🏗️ Architecture
CSV Data → pandas → Text Formatting → HuggingFace Embeddings → FAISS Vector Store
↓
User Question → Query Rewriting → FAISS Similarity Search → Groq LLama → Answer

---

## 📊 Dataset

| File | Description | Rows |
|---|---|---|
| WorldCups.csv | Tournament summaries 1930-2014 | 20 |
| WorldCupMatches.csv | All match results | 852 |
| WorldCupPlayers.csv | All player appearances | 37,784 |

---

## 🛠️ Tech Stack

| Layer | Technology |
|---|---|
| Data Processing | Pandas |
| Embeddings | HuggingFace sentence-transformers/all-MiniLM-L6-v2 (384-dim, local) |
| Vector Store | FAISS |
| LLM | Groq LLama 3.1 8B Instant |
| RAG Framework | LangChain |
| UI | Streamlit |
| Query Rewriting | Groq LLama 3.1 |

## 🚀 How To Run

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/worldcup-rag-agent.git
cd worldcup-rag-agent
```

### 2. Create virtual environment
```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Set up API keys
```bash
cp .env.example .env
```
Add your Groq API key to `.env`:

GROQ_API_KEY=your_groq_api_key_here

Get free Groq API key at: https://console.groq.com

### 5. Run ingestion pipeline
```bash
python ingest.py
```
This reads the CSV files, creates embeddings locally using HuggingFace,
and saves the FAISS vector store to disk.

Expected output:
Loading CSVs...
Tournaments loaded: 20
Matches loaded: 852
Formatting documents...
Total documents created: 872
Loading embedding model...
Creating embeddings and saving to FAISS...
Vector store saved to disk!
Ingestion complete!

### 6. Launch the chat app
```bash
streamlit run app.py
```
Opens at http://localhost:8501

## 🧠 Key Concepts Implemented

**RAG Pipeline**
Complete indexing and retrieval pipeline — ingestion transforms raw CSV 
data into semantically searchable vector embeddings, retrieval finds 
relevant context before generating answers.

**Local Embeddings**
Uses HuggingFace sentence-transformers running locally — zero API calls, 
zero cost, no rate limits during ingestion of 37,784 rows.

**Query Rewriting**
Handles follow-up questions by rewriting them into standalone queries 
before FAISS search, solving context-unaware retrieval.

**Chunking Strategy**
Each CSV row is formatted as one complete document rather than 
splitting fields separately — prevents over-chunking and reduces 
embeddings from 4,572 to 872.

## 🔄 Two Approaches

This project was intentionally built two ways:

**Approach 1 — No Code (n8n)**
Visual drag-and-drop workflow for understanding RAG concepts:
CSV → Edit Fields → HuggingFace Embeddings → Simple Vector Store → AI Agent

**Approach 2 — Python (this repo)**
Production implementation with full control over every component.
